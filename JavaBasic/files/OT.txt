# 프로그래밍이란 무엇인가
-프로그래밍 언어를 사용해 프로그램을 만드는 것
-프로그램을 만드는 사람을 "프로그래머"라고 한다. 
-프로그래밍 언어를 통해 만든 결과물을 "소스코드"라고 부른다.

# 프로그래밍 언어란?
- 컴퓨터는 실제로 0과 1밖에 알아듣지 못한다. 
- 전기가 흐르지 않으면 0, 흐르면 1
- 0과 1만 가지고 프로그램을 만들기는 너무 힘들기 때문에 프로그래밍 언어가 등장.
- 프로그래밍 언어는 사람이 알아듣기 편한 영어와 컴퓨터가 알아듣기 편한 0과 1의 중간 지점이다. 
- 프로그래머가 작성한 소스코드를 0과 1로 번역하는 과정을 "컴파일(compile)"이라고 한다. 
- 언어마다 0과 1에 가까울수록 컴퓨터가 해석하기 편한 "저급 언어 low level"
  (C언어, C++ 로켓도 쏨, 게임같은 건 오래걸림 성능이 좋아야 해서)
- 프로그래머가 작업하기에 편한 프로그래밍 언어를 "고급언어"라고 한다. 
  (JAVA, Python)
- 저급언어는 개발속도가 느리고 성능이 매우 좋다.(성능이 좋으니까 느림)
- 고급언어는 개발속도가 빠르고 성능이 상대적으로 떨어진다.
   (개발속도가 빠르다는 건 인건비가 상대적으로 저렴하다는 뜻. 웹페이지 같은 건 뚝딱)


# 컴파일러 
- 작성한 소스코드를 컴파일 하는 프로그램
- 컴파일 진행 결과로 실행 파일을 생성한다. 
- 작성된 소스코드를 ☆한꺼번에 번역☆하여 실행파일을 생성한다.
- 컴파일러를 사용하는 언어를 "컴파일 언어"라고 부른다. 
  C, C++, JAVA, ...

# 인터프리터
- 작성된 소스코드를 ☆한줄씩☆ 해석하여 실행하는 프로그램
- 인터프리터를 사용하는 언어를 "스크립트 언어"라고 부른다. 
  Python, Javascript, ... 
(별도의 컴파일 과정이 필요 없음)

# JDK (Java Development kit 자바 개발 키트)
- 자바로 프로그램을 개발하기 위해 필요한 도구들의 모음
- 대표적으로 자바 컴파일러(javac), 자바 실행환경(jre), ... 등이 포함돼 있다. 

#JRE (Java Runtime Environment , 자바 실행 환경)
- 자바로 만든 프로그램을 실행하기 위해 필요한 프로그램 (롤 깔 떄 다이렉트X 같은 거 설치하잖아 왜 까냐면 다이렉트X 위에서 개발됐기 때문에)
- JDK 안에 포함돼 있지만, 따로 JRE만 설치할 수도 있다.


# IDE(Integrated Development Environment, 통합 개발 환경)
- 프로그램을 개발할 때 필요한 프로그램들을 하나로 묶어놓은 프로그램
- Java의 IDE : Eclipse, IntelliJ, ... 등이 있다. 
- C, C++의 IDE : Visual Studio, ...
